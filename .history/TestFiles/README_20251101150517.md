# TestFiles Directory

This directory contains all temporary files, test scripts, and debug utilities that are safe to delete without breaking the main system.

## Contents

### Test Scripts
- `test_*.py` - Various unit tests and integration tests
- `debug_*.py` - Debug utilities and diagnostic scripts

### Temporary Data Files
- `*.json` files with timestamps or "test"/"debug" in the name
- Backup files and experimental data

### Demo and Reference Files
- Demo scripts and reference implementations
- Sample output files from testing

### Safe to Delete
All files in this directory are:
- ✅ Safe to delete at any time
- ✅ Not used by the production system
- ✅ Temporary or for testing purposes only
- ✅ Can be regenerated if needed

### Core System Files (NOT in TestFiles)
The following files remain in the root directory as they are part of the core system:
- `app.py` - Main Flask application
- `assessment_engine.py` - Official LSPU assessment system
- `enhanced_assessment_engine.py` - Enhanced assessment system
- `semantic_engine.py` - Semantic analysis engine
- `database.py` - Database management
- `utils.py` - Core utilities
- `pds_extractor.py` - PDS extraction system
- Configuration files (`.env`, `requirements.txt`, etc.)
- Production schemas and templates

## Cleaning Up
To clean this directory:
```bash
# Remove all files in TestFiles (safe operation)
Remove-Item "TestFiles\*" -Recurse -Force

# Keep only this README
# The README will help identify the purpose of this directory
```
