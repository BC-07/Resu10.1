# Semantic Scoring System - Quick Reference

## What Is It?
An AI-powered system that evaluates how well candidates match job requirements based on meaning, not just keywords.

## Key Files
- `semantic_engine.py` - Core AI engine
- `enhanced_assessment_engine.py` - Integration with existing system

## Database Changes
Three new columns added to `candidates` table:
- `semantic_score` (0-100) - Final score
- `semantic_breakdown` (JSON) - Detailed analysis
- `semantic_updated` (timestamp) - Last update time

## How to Use

### Assess One Candidate
```python
from enhanced_assessment_engine import EnhancedUniversityAssessmentEngine

engine = EnhancedUniversityAssessmentEngine()
result = engine.assess_candidate_enhanced(candidate_data, job_data)
print(f"Score: {result['semantic_score']}")
```

### Assess Multiple Candidates
```python
results = engine.batch_assess_candidates(candidates_list, job_data)
```

## Common Queries

### Top 10 candidates by semantic score
```sql
SELECT name, semantic_score FROM candidates 
WHERE semantic_score IS NOT NULL 
ORDER BY semantic_score DESC LIMIT 10;
```

### Candidates with high education match
```sql
SELECT name FROM candidates 
WHERE semantic_breakdown->>'education_relevance' > '0.8';
```

## Scoring Components
1. **Education** (35%) - Degree/background match
2. **Experience** (45%) - Work history relevance  
3. **Skills** (15%) - Technical/soft skills alignment
4. **Quality** (5%) - Content completeness bonus

## Customization

### Change scoring weights (semantic_engine.py)
```python
weights = {
    'education_relevance': 0.35,
    'experience_relevance': 0.45, 
    'skills_relevance': 0.15,
    'overall_quality_bonus': 0.05
}
```

### Change AI model (semantic_engine.py)
```python
# Fast, lightweight (current)
model_name = 'all-MiniLM-L6-v2'

# Higher accuracy, slower
model_name = 'all-mpnet-base-v2'
```

## Performance
- **Speed**: ~0.035 seconds per assessment
- **Memory**: ~200MB for AI model
- **Caching**: Enabled for faster repeat assessments

- Check if content is relevant
- Lower similarity threshold in semantic_engine.py
- Review semantic_breakdown for details
